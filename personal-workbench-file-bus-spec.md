# 个人工作台 V3.1「文件总线」规格书（Round 6）

> 产品经理产出 · 2026-06-10
> 前置依赖：V3 Phase 2（任务舱）验收通过后实施
> 改造对象：`personal-workbench/main.js`、`renderer.js`、`preload.js`、`style.css`、`index.html`

---

## 0. 背景与目标

用户的真实工作流中，文件在多个标签页之间靠「下载到下载目录 → 切页 → 上传时翻目录找文件」人肉搬运：

- 豆包生成的封面图 → 下载 → 裁切脚本去水印 → 上传公司平台
- 浏览器插件测试对话 → 下载 → 上传评估页
- 评估报告 → 下载 → 喂回 Hermes 修改

**目标：让活动任务期间的所有文件自动汇聚到任务文件夹，并能零成本流向任何标签页的上传框。** 即"标签页之间的文件流通"。

现有基础（直接泛化，不重复造轮子）：
- 任务专属文件夹 `temp/tasks/{taskId}_{school}_{course}/`
- 评估页下载捕获 `handleDownloadCompleted`（重命名归档 eval_report）
- 评估页上传注入 `setupWebviewUploadInterceptor`
- 任务舱产物列表（Phase 2 交付）

---

## 1. 功能一：全局下载捕获

### 行为
- **有活动任务时**（`pipelineState.active`）：任意 webview 触发的下载，一律落到当前任务文件夹，不再进系统下载目录。
- 命名规则：保留原文件名；与现有文件重名时追加 ` (2)`、` (3)`。
- 现有的 eval_report 特殊归档逻辑保持优先（先判断是否评估报告，否则走通用落盘）。
- **无活动任务时**：行为与现状一致（系统默认下载目录），不做改动。
- 每次捕获后发 IPC 通知 renderer，任务舱托盘即时刷新 + `showToast("已捕获到任务文件夹: xxx")`。

### 实现要点
- `main.js` 中 `session.on("will-download")`（或现有下载监听处）统一拦截，`downloadItem.setSavePath()` 指向任务文件夹。
- 活动任务的文件夹路径由 renderer 在任务 start/resume/pause/finish 时通过 IPC 同步给 main 进程（新增 `tasks:set-active-folder`，传 null 表示无活动任务）。
- 路径校验沿用 `tasks:open-folder` 的同款防穿越逻辑（必须位于 `temp/tasks` 下）。

## 2. 功能二：任务文件托盘（任务舱产物区升级）

### 行为
- 任务舱「任务产物」区从"固定三行"升级为**实时托盘**：列出任务文件夹下所有文件（含子文件夹一层即可，不递归深层）。
- 每个文件行：类型图标（按扩展名：图片/JSON/PDF/Word/其他）+ 文件名 + 大小/修改时间 + 悬停操作区。
- 悬停操作（图标按钮）：
  1. **打开**（系统默认程序）
  2. **在资源管理器中定位**
  3. **复制绝对路径**（喂给 Hermes 场景高频使用）
  4. **裁切去水印**（仅图片文件显示，见功能四）
  5. **删除**（带确认）
- 原有三个"关键产物"（任务文档/dialogue.json/eval_report）置顶并保留就绪/等待状态徽章；其余文件按修改时间倒序排在下方。
- 文件夹变化实时刷新：main 进程 `fs.watch` 活动任务文件夹，debounce 500ms 推送 IPC。

### 实现要点
- 新增 IPC：`tasks:list-files`（返回 name/size/mtime/isDirectory）、`tasks:file-action`（open/reveal/delete，全部走防穿越校验）；复制路径在 renderer 用 clipboard 即可（main 返回绝对路径列表时已带）。
- preload 只暴露这几个白名单方法，不暴露通用 fs 能力。

## 3. 功能三：全局上传注入（核心，"文件流通"的落点）

### 行为
- **有活动任务时**：在任意 webview 中点击文件上传控件，不再直接弹系统文件选择器，而是先弹**工作台文件选择浮层**：
  - 浮层列出任务文件夹内全部文件（多选，复用托盘的列表数据与图标）
  - 底部两个按钮：「注入所选文件」/「改用系统选择器」（fallback，体验不降级）
  - ESC 或点击遮罩 = 取消本次选择（等同用户取消文件选择）
- **无活动任务时**：不拦截，行为与现状一致。

### 实现要点
- 泛化现有 `setupWebviewUploadInterceptor`：从"仅评估页 + 固定注入两个文件"改为"所有 webview + 弹浮层让用户选"。评估页在流水线"评估上传"步骤保留**自动注入**（现有行为优先，不弹浮层）。
- 技术路径沿用现有拦截器机制（webview 内 `file-chooser` 拦截/注入实现已验证可行），main 进程负责把所选绝对路径注入。
- 浮层 UI 放 renderer 层（普通 DOM dialog，z-index 高于一切，复用 V3 设计令牌），**不要做成绝对定位叠 webview 的常驻元素**，弹出期间临时显示即可（modal dialog 不受 webview 原生窗口遮挡问题影响——若实测被遮挡，参考 Round 4 经验改为临时隐藏 webview 指针事件 + 全屏遮罩）。

## 4. 功能四：裁切去水印集成

### 行为
- 托盘中图片文件（png/jpg/jpeg/webp）的「裁切去水印」按钮：点击后从图片**底部裁掉 N 像素**，生成同目录新文件 `原名_cropped.扩展名`，原文件保留。
- N 默认 100，可在「⋯ 更多 → 扩展设置」同级新增的「工作台偏好」里配置（裁切方向 底部/顶部/右侧 + 像素数，默认 底部/100）。
- 完成后 toast + 托盘自动刷新（fs.watch 会触发）。

### 实现要点
- main 进程用 Electron 自带 `nativeImage`：`createFromPath` → `crop({x:0, y:0, width, height: h-N})` → `toPNG()/toJPEG(90)` 写回。**不引入新依赖**。
- 偏好持久化到 Electron userData（沿用现有设置存储方式）。

## 6. 验收标准

1. 执行任务后，在豆包/任意标签页下载文件 → 文件直接出现在任务文件夹与托盘，系统下载目录无副本；无活动任务时下载行为不变。
2. 托盘实时反映文件夹内容（外部用资源管理器增删文件，500ms 内托盘同步）；打开/定位/复制路径/删除五个操作全部可用且不能越出 temp/tasks。
3. 活动任务期间在任意网页点上传 → 弹工作台浮层，多选注入成功；「改用系统选择器」可正常 fallback；评估页流水线自动注入行为不变。
4. 图片裁切：默认底部 100px，生成 `_cropped` 新文件，原图保留；偏好修改后生效。
5. eval_report 捕获、dialogue.json 归档、Hermes 路径注入等既有流水线行为零回归。
6. `npm run check` 通过；preload 新增 API 仅白名单四个，无通用 fs 暴露。

- 功能一+二 一个 commit，功能三 一个 commit，功能四 一个 commit（便于回滚）。
- 每个 commit 后 `npm run check`，全部完成后向产品经理回报验收自检。
