# 个人定制工作台重构与内置助手功能恢复技术规格书 (Round 4 - 任务工作流与蓝线修复)

根据用户的最新反馈，我们需要对工作台菜单布局、分屏溢出 Bug、任务状态栏和任务文件夹工作流进行细节修正。本规格书为 Codex 提供了具体的实现指南：

---

## 1. 核心界面交互与布局修复规格

### 1.1 移除左下角“扩展设置”与原生菜单栏
* **移除 HTML 节点**：彻底删除 `index.html` 中 `footer` 内的 `#settings-button`（扩展设置按钮）及其所有相关样式，保持侧边栏极简。
* **隐藏原生菜单**：在 `main.js` 中设置 `mainWindow.setMenuBarVisibility(false)` 以隐藏 Electron 自带的窗口顶部菜单。

### 1.2 HTML 自定义菜单栏与居中显示“正在进行任务”横幅
* **界面结构**：在 `.workspace` 最上方定义 `#app-menu-bar`（HTML 自定义菜单栏）。
* **任务横幅触发与显示规范**：
  - **默认状态**：未执行任务时，中间的任务横幅 `正在进行任务` 区域 **完全隐藏/不显示**（不显示“正在进行任务: 无”等占位字符），保持菜单栏中间空白。
  - **运行状态**：只有当用户在任务列表中点击“执行”按钮后，菜单栏正中间才**动态显现**横幅：`🏃 正在进行任务: [学校] - [课程] (任务类型)`。
  - 提供了“结束”按钮，点击可一键清空自动化状态，并重新隐藏横幅。
* **直接点击顶级菜单**：
  - 左侧的“任务”和“终端”按钮为直接点击的顶级按钮。点击“任务”直接开启/关闭任务弹窗；点击“终端”直接开启/关闭底部终端。无多余的二级下拉菜单。
  - 菜单栏的“工作台”、“编辑”、“视图”仍保留悬浮触发的二级下拉菜单。

### 1.3 彻底修复蓝线拖拽 Bug (WebView 覆盖问题)
* **Bug 根源**：在 Electron 中，`<webview>` 标签渲染为原生的子窗口。当面板（如扩展面板）采用绝对定位且 resizer 设为 `left: -5px` 时，resizer 叠在 Webview 边缘的 5px 区域会被 Native Webview 窗口无条件覆盖，导致鼠标无法触发 hover 和 drag 事件。
* **重构方案 (Flex 布局 Sibling 兄弟元素)**：
  - 将 resizer 从面板子元素中剥离，转而作为 **Webview 与面板之间的独立 Flex 兄弟元素** 放置在 HTML 结构中：
    ```html
    <div class="tab-viewport active">
      <webview class="tab-webview"></webview>
      <div class="tab-extension-resizer" title="拖动调整宽度"></div>
      <div class="tab-extension-panel">
        <div class="tab-extension-content">
          <!-- 包含 header 和 body -->
        </div>
      </div>
    </div>
    ```
  - **CSS 样式定义**：
    - `.tab-extension-resizer`：宽度设为 `4px`，默认背景透明，鼠标 hover 时背景亮起 `var(--accent-color)`，光标设为 `ew-resize`。
    - 因为它是 Flex 容器下的独立兄弟元素，Webview 的原生窗口会自动排列在它的左侧，**绝对不会发生重叠遮挡**！
    - 当面板关闭时（宽度为 0），resizer 仍然作为兄弟元素排在 Webview 右侧，用户可随时 hover 并拖拽拉开面板。
  - 同理，原生助手面板的 resizer 也需按照此 Sibling 结构进行重构。

## 2. 任务专属文件夹与 Hermes 闭环工作流规格

为了打通“测试 -> 评估 -> 诊断”的完整闭环，避免频繁上传下载，设计如下自动化任务工作流：

### 2.1 自动创建任务专属文件夹
* **目录结构**：当用户在任务列表中点击“执行”时，工作台自动在本地项目根目录的 `temp/tasks` 下创建该任务的专属文件夹：
  - 路径：`temp/tasks/{taskId}_{school}_{course}/` （对非安全文件名字符进行过滤替换）。
* **文件归档**：
  - **任务文档**：该文件夹中用于存放该任务的初始任务文档（用户可按需手动放入该文件夹）。
  - **测试对话**：测试完成后，助手生成的测试对话记录直接保存至该专属文件夹下，命名为 `dialogue.json`。

### 2.2 一键注入评估界面
* 当自动化流程导航至评估界面（`https://www.wl363eval.top/`）时：
  - 自动将任务专属文件夹下的 **任务文档**（如有）和最新生成的 **测试对话记录 (`dialogue.json`)** 一起通过 Webview 的文件选择拦截器自动注入到评估页面的上传输入框中。

### 2.3 自动拦截与捕获评估报告
* 评估完成后，当用户触发下载报告或系统自动下载时，工作台的主进程拦截下载请求：
  - 自动将评估报告（PDF/HTML）重命名并移动保存到当前任务的**专属文件夹**下，命名为 `eval_report.pdf` (或 `eval_report.html`)。

### 2.4 一键加载文件地址至 Hermes 会话框
* **功能入口**：在工作台顶部栏或助手面板上，提供一个“**复制路径至 Hermes**”的按钮（仅在当前任务完成评估报告捕获后激活）。
* **动作规范**：
  - 点击后，自动读取该任务文件夹下 `dialogue.json` 和 `eval_report.pdf` 的**本地绝对路径**。
  - 格式化生成文本：
    ```text
    测试对话记录地址：【测试对话的本地绝对路径】
    评估报告地址：【评估报告的本地绝对路径】
    ```
  - 自动切换至 Hermes 标签页，获取其 Webview 的输入框（`textarea` 或 `[contenteditable]`），将上述生成的文本**灌入到输入框中，但先不要发送**，保留在输入框中等待用户确认。
